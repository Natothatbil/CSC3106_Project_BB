# Appendix: AI-Use Declaration (Part 1)

**Tool used:** Claude (Anthropic), via Claude Code.

**What we used it for:** Working out what the different `sshd`/`sudo`/
`CRON` message formats in the log actually meant before we started
writing the parser; drafting `analysis.py` itself; suggesting a few
analysis angles we hadn't thought of (in particular, checking whether any
IP's failed attempts were immediately followed by a success, which is
where the `deploy` compromise finding came from); and drafting the prose
in the report, this declaration, and the risk matrix.

**How we checked/corrected what it gave us:** Every number in the report
came from actually running `analysis.py` against our extract and reading
the output files, not from asking the model to describe the log. Worth
being specific about one thing: an early version of the "consecutive
failed attempts before success" metric turned out to be broken. It
summed failed attempts across multiple unrelated days instead of looking
at a single attack burst, which produced a misleading number (1,120,
which we then caught and fixed to the correct 117-in-34-minutes figure
before using it anywhere). We only trusted that number once we'd traced
through the logic and confirmed it against the raw log lines by hand. We
also spot-checked the regex patterns against sample log lines (e.g. the
`Failed password for invalid user X` vs `Failed password for X`
difference, and the break-in-warning line format) rather than assuming
they were right the first time.

**How much of the submitted work was substantially shaped by AI:** The
first drafts of `analysis.py`, the report prose, the risk matrix, and
this declaration were all produced with AI assistance. What we decided
ourselves: which asset to focus the risk matrix on, which risks to
prioritise and why, and reviewing every figure/table against the raw log
before it went into the report.

**How we made sure the final claims are Python-verified:** Nothing in
the report is taken from the AI's own description of what's in the log.
Every claim traces back to a file in `output/`, generated by running:

```
python analysis.py ../1_auth.log.txt --outdir output
```

Anyone marking this can rerun that command and get the same numbers.
